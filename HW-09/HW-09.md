# Домашнее задание

## Архитектура для высоких нагрузок и Real-time обработки

**Цель:**  
В этом ДЗ вам нужно оптимизировать архитектуру под жёсткие требования производительности (High Load). Результатом выполненного задания станет оптимизированная архитектурная схема с кэшированием и асинхронностью.

## Описание/Пошаговая инструкция выполнения домашнего задания:

**Задача:**  
Ваш сервис вырос. Теперь нагрузка 10,000 RPS, а требование по Latency < 200ms.

**Шаги выполнения:**

1. **Caching Strategy:** Внедрите кэширование. Где? (CDN, API Gateway, Redis для эмбеддингов - Semantic Cache). Опишите алгоритм Semantic Cache.

2. **Scaling:** Предложите схему горизонтального масштабирования Inference-подов (KEDA autoscaling по метрике очереди запросов).

3. **Async Processing:** Для тяжелых запросов предложите переход на асинхронную модель (Queue-based).

4. **Global Distribution:** (Опционально) Если пользователи по всему миру, как используете Geo-DNS и Edge computing?

## Формат сдачи:

- Схема High-Load архитектуры с пояснительной запиской по выбранным паттернам оптимизации.

## Вспомогательный материал:

- Semantic Caching
- Документация KEDA

Если у вас возникнут вопросы, вы можете задать их в чате проверки домашнего задания или в группе в Telegram.

## Критерии оценки:

**Статус «Принято», если:**

- **Адекватность мер:** Semantic Cache реально снижает нагрузку на LLM.
- **Масштабируемость:** Решение выдержит рост нагрузки.
- **Latency:** Предложенные меры (кэш, edge) направлены на снижение задержек.

## Компетенции:**

Применять сложные архитектурные паттерны для решения задач масштабирования, потоковой обработки данных (real-time) и построения безопасных гибридных сред:

- аргументированно обосновывать выбор платформы оркестрации (Serverless или Kubernetes)
- проектировать EDA-архитектуру (Event-Driven Architecture) для реактивных систем
- проектировать архитектуру высоконагруженных сервисов с требованиями к низкой задержке (кэширование, Edge computing)
