# Домашнее задание

## Автоматизация поставки: IaC, CI/CD и MLOps-конвейеры

**Цель:**  
В этом ДЗ вам нужно спроектировать автоматизированный конвейер доставки кода и моделей в продакшн. Результатами выполненного задания станут схема CI/CD + MLOps пайплайна и стратегия релиза.

## Описание/Пошаговая инструкция выполнения домашнего задания:

**Шаги выполнения:**

1. **Infrastructure as Code:** Опишите (псевдокодом Terraform или текстом), какие ресурсы будут подниматься автоматически (K8s cluster, S3 buckets).

2. **CI/CD Pipeline:** Нарисуйте схему пайплайна: Commit -> Build Docker -> Unit Tests -> Deploy to Staging -> E2E Tests -> Deploy to Prod.

3. **MLOps Integration:** Покажите, как этот пайплайн связан с обучением модели. (Например: Обновился датасет -> Airflow Trigger -> Retrain -> Model Registry -> Trigger CI/CD).

4. **Release Strategy:** Опишите стратегию Canary Deployment. Как будете переключать трафик? (1% -> 5% -> 100%). Какие метрики отката (rollback) будете использовать?

## Формат сдачи:

- Схема пайплайнов (CI/CD + MLOps) и текстовое описание стратегии релиза.

## Вспомогательный материал:

- Architectural Overview
- Canary Release

Если у вас возникнут вопросы, вы можете задать их в чате проверки домашнего задания или в группе в Telegram.

## Критерии оценки:

**Статус «Принято», если:**

- **Интеграция:** Связь между кодом приложения и артефактом модели (Model Registry).
- **Безопасность релиза:** Наличие этапов тестирования и стратегии Canary/Blue-Green.
- **Автоматизация:** Минимум ручных действий в схеме.

## Компетенции:

Планировать и организовывать надежную инфраструктуру для AI-систем, включая настройку процессов автоматизированного развертывания (CI/CD) и обеспечение отказоустойчивости:

- писать Terraform-конфигурации для развертывания базовой инфраструктуры (VPC, Kubernetes-кластер, S3)
- проектировать схемы MLOps-пайплайнов для RAG-систем с автоматическим обновлением векторной базы
- разрабатывать планы релиза для AI-сервисов с использованием стратегии Canary
